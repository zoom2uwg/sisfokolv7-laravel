### 🏗️ Arsitektur Keputusan Epic 8

1. **Multi-Tenant Isolation:** Semua query otomatis di-scope ke `tenant_id` via trait `BelongsToTenant`
2. **QR Scanner Anti-Spam:** Pemeriksaan duplikasi dalam `DB::transaction()` untuk mencegah race condition
3. **Approval Workflow State Machine:** Hanya izin berstatus `pending` yang dapat diproses
4. **Policy-Based Authorization:** Seluruh controller menggunakan `Gate::authorize()` — tidak ada proteksi middleware role saja
5. **Dynamic Rules Engine:** Jam masuk dan toleransi terlambat dibaca dari `tenant_settings`, bukan hardcoded

#### 3. Core Features

**A. Event Subscribers (Kunci Integrasi):**

1. **EvaluationFrameworkSubscriber:**
   - Mendengarkan event: `Evaluation.ResolveFramework`
   - Trigger: Ketika modul Evaluation butuh framework KI/KD untuk mapel tertentu
   - Action: 
     - Cek `mapel.kurikulum_id`
     - Resolve `struktur_kurikulum` + `komponen_kompetensi` berdasar jenjang/kelas
     - Isi `$event->framework` dengan data KI/KD atau CP

2. **RaporSectionSubscriber:**
   - Mendengarkan event: `Raport.RenderSection`
   - Trigger: Ketika modul Rapor generate PDF
   - Action: Inject section "Capaian Kompetensi" ke dalam rapor
